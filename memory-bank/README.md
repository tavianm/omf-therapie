# Memory Bank

## Purpose

This memory bank serves as a centralized knowledge repository for the OMF Therapie project. It maintains context, decisions, conventions, and architectural information to ensure consistency and continuity throughout development.

## Structure

### Core Files

- **project-overview.md** - High-level project description, goals, and context
- **technical-stack.md** - Technology choices and rationale
- **architecture.md** - System architecture and design patterns
- **conventions.md** - Coding standards and project conventions
- **decisions.md** - Architectural Decision Records (ADRs)
- **active-context.md** - Current work in progress and recent changes

### How to Use

1. **Before Starting Work**: Review relevant memory bank files to understand context
2. **During Development**: Update active-context.md with significant changes
3. **After Major Decisions**: Document in decisions.md with rationale
4. **Regular Updates**: Keep project-overview.md and conventions.md current

## Maintenance

- Review and update files regularly
- Archive outdated information
- Keep documentation concise and actionable
- Date all significant updates
